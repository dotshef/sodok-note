# Phase 1: 프로젝트 기반 세팅 + 인증

## 목표
Supabase 연동, DB 스키마 생성, 인증(로그인/회원가입), 멀티테넌시 기본 구조 확립

## 선행 조건
- Next.js 프로젝트 초기화 완료
- Supabase 프로젝트 생성 완료

## 작업 목록

### 1-1. Supabase 연동 및 환경 설정
- [ ] Supabase 클라이언트 설정 (`@supabase/ssr`)
- [ ] 환경변수 설정 (SUPABASE_URL, SUPABASE_ANON_KEY)
- [ ] 서버/클라이언트 Supabase 클라이언트 유틸 생성

### 1-2. DB 스키마 생성
- [ ] `tenants` 테이블 생성
- [ ] `users` 테이블 생성 (email + password_hash 직접 관리)
- [ ] `facility_types` 마스터 테이블 생성 + 시드 데이터 삽입 (13개 시설 유형)
- [ ] `clients` 테이블 생성
- [ ] `schedules` 테이블 생성
- [ ] `visits` 테이블 생성
- [ ] `certificates` 테이블 생성
- [ ] 서버 사이드 tenant_id 기반 멀티테넌시 유틸 함수 작성 (service_role 키 사용)

### 1-3. 인증 (Auth)
- [ ] 회원가입 페이지 — 이메일/비밀번호 + 업체 정보 입력
- [ ] 회원가입 시 `tenants` + `users`(admin) 자동 생성
- [ ] 로그인 페이지 — 좌측 브랜드 소개 + 우측 로그인 폼 (2분할 레이아웃)
- [ ] "로그인 상태 유지" 체크박스
- [ ] "비밀번호 찾기" 링크 + 비밀번호 재설정 플로우
- [ ] 로그아웃 기능
- [ ] 인증 미들웨어 — 미로그인 시 리다이렉트
- [ ] 세션 관리 (JWT + httpOnly 쿠키 직접 구현)

### 1-4. 레이아웃 기본 구조
- [ ] 좌측 사이드바 레이아웃 셸 (데스크톱 기본)
- [ ] 사이드바 메뉴: 대시보드 / 캘린더 / 고객 관리 / 증명서 / 기사 관리 / 설정
- [ ] 상단 헤더 — 페이지 타이틀 + 관리자 토글
- [ ] 모바일 반응형 (사이드바 → 하단 탭 또는 햄버거 메뉴)

## 산출물
- 로그인/회원가입 동작
- DB 테이블 7개 + 서버 사이드 권한 체크
- 앱 레이아웃 셸
